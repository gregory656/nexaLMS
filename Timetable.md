# Timetable Module — Implementation Summary ✅

## Overview

The NexaLMS Timetable module is a **constraint-satisfaction scheduling system** — not a simple table filler. Admin configures constraints first, then the generator searches for a valid schedule using backtracking.

---

## What Was Built

### 1. Database Migration (`00004_timetable_module.sql`)

| Change | Details |
|--------|---------|
| `subjects.lessons_per_week` | Default weekly lesson requirement per subject (1–20) |
| `teacher_subject_assignments.lessons_per_week` | Per teacher-subject-class weekly lessons |
| `timetable_settings` | Admin configuration table (working days, periods, breaks, limits) |
| `timetables` extended | `version`, `term_name`, `status`, `generated_at`, `generated_by`, `settings_snapshot` |
| `timetable_entries` extended | `period_number`, `day_of_week` now 0–6 (Mon–Sun) |
| `timetables.term_id` | Made nullable (term_name used as label) |

### 2. Timetable Page (`/academics/timetable`)

Three-tab workflow:

#### Setup Tab
- **Academic Year** — auto-filled from current academic year dashboard
- **Term** — Term 1 / 2 / 3 dropdown
- **Timetable Name** — e.g. "Term 2 2026"
- **Working Days** — presets (Mon–Fri, Mon–Sat, Tue–Sat) + custom day picker
- **Lesson Configuration** — duration, periods/day, start/end time, teacher/class daily limits
- **Break Configuration** — multiple breaks (Tea, Lunch, etc.) — blocked from lessons
- **Data Sources** — live counts of classes, teachers, assignments

#### Generate Tab
- Validates prerequisites (classes, subject combinations, settings)
- Progress UI: "NexaLMS is generating your timetable..." with percentage bar
- Backtracking constraint solver places all required lessons
- Creates versioned timetable (V1, V2, V3...)
- Version history table with view/delete

#### Preview Tab
- **Master Timetable** — all classes in grid view
- **Teacher Timetable** — filter by teacher
- **Class Timetable** — filter by class
- **Publish** — marks version as published (teachers/students see latest published)
- **Download PDF** — opens print-ready page with school watermark, header, footer

### 3. Constraint Engine (`src/lib/timetable/generator.ts`)

Constraints enforced:
- Teacher cannot teach two classes simultaneously
- Class cannot have two subjects in one period
- Breaks are excluded from available slots
- Teacher daily max lessons respected
- Class daily max lessons respected
- Exact weekly lesson count per assignment (e.g. Math = 6, not 5 or 7)

Algorithm: backtracking with slot shuffling for variety.

### 4. PDF Export (`src/lib/timetable/pdf.ts`)

- School name header (uppercase)
- Timetable type title (MASTER / TEACHER / CLASS)
- Filter name (teacher or class name)
- Term + year subtitle
- Grid table with all lessons
- School logo/watermark (centered, light opacity)
- Footer: "Generated by NexaLMS | date | Version V{n}"
- Filenames: `master_timetable_V1.pdf`, `teacher_stephen_otieno_V2.pdf`, `form2_white_V3.pdf`

### 5. Subject Combinations (`Staff & Teachers` page)

New **Subject Combinations** tab:
- Table: Teacher | Subject | Class | Lessons/Week | Actions
- Add/Edit/Delete combinations
- Used as input for timetable generator

**New Staff** modal now requires at least 1 subject combination:
- Subject dropdown (from Subjects module)
- Class dropdown (from Streams & Classes)
- Lessons per week input

### 6. Subjects — Weekly Lessons

Subjects page now includes **Lessons/Week** column and form field — default for new teacher assignments.

## Recommended Workflow

1. **Setup academic structure** — Streams, Grades, Classes, Academic Year
2. **Add subjects** — with weekly lesson counts
3. **Add staff** — with at least one subject combination each
4. **Configure timetable** — Setup tab (days, periods, breaks, limits)
5. **Generate** — click Generate Timetable
6. **Preview** — Master / Teacher / Class views
7. **Publish** — make live for school
8. **Download PDFs** — individual or bulk via print dialog

## Versioning

Each generation creates a new version (V1 → V2 → V3). Old versions remain for reference unless deleted. Only one version can be **published** at a time.
